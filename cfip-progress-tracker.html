Phase 6 — Real SMART on FHIR integration
in progress
Goal

Make CFIP a genuine SMART on FHIR Backend App by wiring real Epic FHIR API calls. JWT auth is registered with Epic — Phase 6 exercises it: authenticate, fetch real patient data, use synthetic overlay only for gaps.

Auth + FHIR fetch flow
Deliverables
  • D1 — JWK Set endpoint: GET /.well-known/jwks.json returns RSA public key (currently 404)
  • D2 — FHIR Client: JWT → token → parallel fetch of 5 resource types, token cached in memory
  • D3 — Orchestrator update: FHIR-first data layer replaces prefetch-only reads
  • D4 — Synthetic overlay switches to gap-fill mode (not replace mode)
  • D5 — Harness animation: "Auth with Epic ✓" + "Fetch FHIR Record ✓" steps
  • D6 — Data source tags on cards: shows exactly what came from Epic vs synthetic
  • D7 — Tests: JWT assertion, token caching, FHIR fetch, 404 graceful, 401 re-auth
Graceful degradation
  • If Epic FHIR call fails — synthetic overlay covers 100%, no card failure
  • Epic sandbox patients won't have PGx / PD-L1 / carboplatin history — synthetic fills these transparently
  • USE_SYNTHETIC_OVERLAY=false in .env enables pure FHIR mode for real hospital deployments
